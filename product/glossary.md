## 1. Bounded Context Overview

Product BC는 **상품 카탈로그(이름·가격·카테고리·상태)의 SSOT**다 (ADR-0013). 재고는 본 BC가 직접 관리하지 않고 **Inventory BC가 SSOT**이며, 본 BC는 Inventory의 변경 이벤트를 받아 `stock` 필드를 캐시 동기화한다. 상품 변경은 Kafka 토픽 `product.item.created` / `product.item.updated`로 발행되며 Search BC가 인덱싱을 위해 소비한다.

---

## 2. Aggregates & Entities

### Product
**Type.** Aggregate
**Definition.** 카탈로그의 단일 상품 단위. 이름·가격·재고 캐시·상태를 보유.
**Lifecycle.** `create(name, price, stock≥0)` → ACTIVE → `update(name?, price?)` 반복 → `deactivate()` → INACTIVE
**Invariants.**
- `name.isNotBlank()` (생성·update 시) — 빈 상품명 금지
- `stock >= 0` (생성·syncStock 시) — 음수 재고 금지
- `deactivate()`는 ACTIVE 상태에서만 가능 — 위반 시 `IllegalStateException`
- `stock` 필드는 **Inventory BC의 캐시**일 뿐 SSOT 아님 (ADR-0013)
**Related events.** emits: `product.item.created`, `product.item.updated` (Kafka)
**Avoid.** Item, SKU, Goods (의미 분화 시 분리)
**Code.** `com.kgd.product.domain.product.model.Product`
**Found in.** `product/domain/product/model/`
**References.** ADR-0013 (Product-Inventory SSOT)

## 3. Value Objects

### Money
**Type.** Value Object (`@JvmInline value class`)
**Definition.** 금액을 표현하는 값 객체. BigDecimal 래퍼.
**Invariants.**
- `amount > BigDecimal.ZERO` — 0 또는 음수 금액 금지
- `plus(other)` 산술 지원
**Avoid.** Amount, Price (의미 분화 시)
**Code.** `com.kgd.product.domain.product.model.Money`

> **Cross-context 차이**: Order BC의 Money는 `amount >= ZERO` (0 허용). 본 BC는 `> ZERO` (0 금지). **의미가 미세하게 다름 — 통합 또는 명확한 분리 결정 필요** (Flagged).

## 6. Policies & Invariants

## 11. Cross-Context Integration

### Emitted Events
- `product.item.created` / `product.item.updated` → **search BC**가 인덱싱

### Consumed Events
- inventory BC의 InventoryEvent (StockReserved/Released/Confirmed/Received) → `InventoryStockSyncConsumer` → `Product.syncStock(availableQty)`

### Shared Terms
| 용어 | 본 BC | 다른 BC | 해결 |
|---|---|---|---|
| **Money** | `amount > ZERO` | order BC: `amount >= ZERO` (0 허용) | **🔴 정의 불일치 — 통합 결정 필요** |
| **Stock** | 캐시된 가용수량 | inventory BC: 권위 보유체 (Inventory) | 본 BC는 캐시, inventory가 SSOT (ADR-0013) |
| **Product** | 카탈로그 권위 | search BC: ProductDocument(읽기 모델) | 본 BC가 SSOT |
| **ProductStatus** | ACTIVE/INACTIVE | order BC의 OrderStatus와 무관 | — |
