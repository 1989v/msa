## Where Blocked

| Item | Root Cause | Duration | Resolution |
|------|-----------|----------|------------|
| **auto-commit hook 의 push 실패 silent** | 35개 commit 동안 hook 메시지는 "auto-committed and pushed" 라고 출력했지만 실제 origin/main 도달 0회. 사용자가 "갱신해줘" 요청 후 marketplace 캐시 조회 시점에야 발견 | ~5분 (디버깅) | `cd ai && git push origin main` 수동 push 시도 → rejection → 다음 항목으로 |
| **origin/main force-push 흔적** | 로컬 8 commit (stale) vs origin 8 commit (post-force-push, 같은 메시지/다른 SHA). `git pull` 이 divergent 에러 | ~3분 | `git fetch` → `git rev-list --left-right` 로 검증 → `git reset --hard origin/main` 캐시는 안전 |
| **첫 auto-commit 이 회사 git identity 로 author** | `d830e57c` (가장 오래된 신규 commit) 이 `gideok.kwon@myrealtrip.com` 으로 author. pre-push hook 거부 | ~2분 | `git rebase --exec 'git commit --amend --no-edit --reset-author' origin/main` 일괄 교정 → push fast-forward |
| **rebase 충돌 (1차 시도)** | `git rebase origin/main` 이 stale commit 8개를 replay → doctor.md/plugin.json 등 conflict | ~2분 | `git rebase --abort` → `git rebase --onto origin/main 59c9941` (boundary SHA) 로 surgical replay |
| **PRD 1차 작성 시 시뮬레이션을 과대 평가** | "Full MVP" 옵션 선택 시 자동으로 시뮬레이션 포함했으나 사용자 핵심 의도는 "합성/충돌 가시화" 였음. 별도 질문 라운드에서 사용자 정정 | ~1라운드 | Section 2 P1/P2 재분리 + 시뮬레이션 → v2 격하. PRD 명시적 `core-intent` 필드 추가 |
