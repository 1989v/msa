# Requirements: Agent Team Visualizer

## 개요
Claude Code에 설치된 플러그인/에이전트 팀을 2D 픽셀 아트 RPG 오피스 스타일로 시각화하는 프론트엔드 서비스.
QJC OS Visualizer 레퍼런스 — 에이전트들이 오피스 환경에서 각자 데스크에 앉아 일하는 모습.

## 레퍼런스 (QJC OS Visualizer 캡처 분석)
- 다크 테마 대시보드
- 좌측 사이드바: 네비게이션 메뉴 (시스템 목록, 아키텍처, 실시간 모니터, 카탈로그, 작업흐름 설계, 사용 통계, 역할 프로파일 등)
- 중앙: 오피스 그리드 뷰 — 픽셀 아트 캐릭터들이 데스크에 배치
- 우측: 에이전트 상세 프로파일 패널
- 전체적으로 2D 탑다운 오피스 게임 느낌

---

## SR-1: 기술 스택
- 기존 `front/` 디렉토리에 프로젝트 생성
- 프레임워크 선택은 구현 시 결정 (React/Vite 권장)
- 단일 페이지 애플리케이션 (SPA)

## SR-2: 에이전트 데이터 소스
- `claude plugins list` CLI 출력을 파싱하거나 API 형태로 제공
- 플러그인 목록, 에이전트 목록, 스킬 목록을 구조화된 JSON으로 변환
- 정적 JSON fallback 포함 (CLI 사용 불가 시)

## SR-3: 계층 구조 시각화
- **플러그인 = 팀** (harness-scaffold, doc-scaffolding, ai-debugger, content-analyzer, private-repo, superpowers 등 11개)
- **에이전트 = 팀원** (implementer, tester, verifier, scaffolding-agent, analyzer-agent 등 17개+)
- 플러그인 팀별 영역 구분, 하위 에이전트를 팀원으로 표시

## SR-4: 픽셀 아트 RPG 캐릭터
- 16~32px 도트 스프라이트 캐릭터
- 각 에이전트별 고유 외형 (역할 반영: 코더, 테스터, 리뷰어, 디버거 등)
- 오피스 데스크에 앉아있는 기본 포즈
- 상태별 애니메이션: idle(대기), working(일하기), thinking(고민)
- 말풍선으로 현재 작업/상태 표시

## SR-5: 대시보드 레이아웃
- **다크 테마** (QJC OS Visualizer와 동일)
- **좌측 사이드바**: 네비게이션
  - 에이전트 목록 (팀별)
  - 시스템 아키텍처 뷰
  - 실시간 상태 모니터
  - 역할 프로파일
- **중앙 메인 뷰**: 오피스 그리드 — 에이전트 캐릭터 배치
- **우측 패널**: 선택한 에이전트 상세 정보 (역할, 도구, 스킬 목록, 상태)

## SR-6: 에이전트 상호작용
- 에이전트 클릭 시 우측 패널에 상세 정보 표시
- 말풍선: 에이전트 역할 설명 또는 현재 상태
- 상태별 애니메이션 전환 (idle ↔ working ↔ thinking)
- 팀(플러그인) 영역 호버 시 팀 정보 표시

## SR-7: 에이전트 프로파일 데이터
각 에이전트가 가져야 할 정보:
- 이름 (예: "Implementer", "Tester")
- 소속 팀/플러그인 (예: "harness-scaffold")
- 역할 설명
- 사용 가능한 도구 목록
- 현재 상태 (idle/working/thinking)
- 캐릭터 스프라이트

---

## Out of Scope
- 실시간 WebSocket 연동 (1차에서는 정적/폴링)
- 에이전트 실행 제어 (UI에서 에이전트 실행하는 기능)
- 모바일 반응형 (데스크톱 우선)
- 백엔드 서비스 신규 개발 (기존 MSA 서비스에 포함하지 않음, 순수 프론트엔드)
