## 1. Bounded Context Overview

Analytics BC는 **이벤트 수집·스코어 산출·메트릭 조회**. ClickHouse 단독 소유 (analytics DB). Kafka Streams로 1시간 윈도우 준실시간 집계. Redis 캐시 (Score TTL 2h, Stats TTL 1h). 이벤트 소비 토픽 `analytics.event.collected`, 발행 토픽 `analytics.score.updated`. ADR-0017.

## 2. Aggregates & Entities

### ProductScore
**Type.** Aggregate (data class — 불변 스냅샷)
**Definition.** Product 의 분석 지표 집계 스냅샷. CTR/CVR + 가중 인기 점수.
**Lifecycle.** `compute(productId, impressions, clicks, orders, normalizer)` → 영속 → 다음 compute 결과로 대체
**Invariants.**
- **Popularity 가중치 공식**: `impressions * 1.0 + clicks * 3.0 + orders * 10.0` (View 1, Click 3, Order 10)
- `ctr = clicks / impressions` (impressions > 0 일 때, 아니면 0.0)
- `cvr = orders / clicks` (clicks > 0 일 때)
- `popularityScore = normalizer(rawPopularity)` (정규화 함수 주입)
- `updatedAt = Instant.now()` 자동
**Code.** `com.kgd.analytics.domain.model.ProductScore`

### KeywordScore
**Type.** Aggregate (data class)
**Definition.** 검색 키워드의 인기·전환 지표.
**Invariants.**
- **Score 공식**: `searchCount * 1.0 + ctr * 5.0 + cvr * 10.0` (Search 1, CTR 5, CVR 10)
- ctr/cvr 동일 0-divide 가드
**Code.** `com.kgd.analytics.domain.model.KeywordScore`

## 4. Domain Services

### ScoreNormalizer
**Type.** Domain Service (object — singleton)
**Definition.** 원시 Score를 0~1 범위로 정규화. min-max + 95-percentile clipping.
**Algorithm.**
```
upperBound = max * 0.95          (95-percentile clip)
if upperBound <= min: return 0.0
clipped = value.coerceIn(min, upperBound)
return (clipped - min) / (upperBound - min)
```
**Why clip at 95%?** outlier(상위 5%) 영향 완화 — 상위 outlier가 normalized score를 압축하는 것 방지.
**Code.** `com.kgd.analytics.domain.model.ScoreNormalizer`

## 5. Domain Events

### ScoreUpdateEvent (Kafka, outgoing)
**Type.** Domain Event
**Definition.** ProductScore 갱신 결과를 외부에 발행.
**Trigger.** Kafka Streams topology 가 1시간 윈도우 마감 시
**Kafka topic.** `analytics.score.updated`
**Code.** `com.kgd.analytics.app.event.ScoreUpdateEvent`
**Consumers.** search BC `ProductScoreUpdateConsumer` → ProductDocument 갱신

## 6. Policies & Invariants

| 출처 | 규칙 |
|---|---|
| ProductScore Popularity 공식 | View:Click:Order = 1:3:10 가중치 (도메인 결정) |
| KeywordScore 공식 | Search:CTR:CVR = 1:5:10 가중치 |
| ScoreNormalizer | 95-percentile clip + min-max normalize |
| `impressions > 0` 가드 | 0-divide 방지 (CTR 계산) |
| `clicks > 0` 가드 | 0-divide 방지 (CVR 계산) |
| 캐시 TTL | Score 2h, Stats 1h (CLAUDE.md) |
| Kafka window | 1h (CLAUDE.md) |

## 7. Ports

### Outbound

#### EventRepositoryPort
**Definition.** 원시 이벤트 저장 (ClickHouse) + 실험 메트릭 쿼리.
**Methods.** `saveEvents(List<AnalyticsEvent>)`, `queryExperimentMetrics(experimentId, startTime, endTime): List<ExperimentMetricRow>`

#### ProductScoreRepositoryPort / KeywordScoreRepositoryPort
**Methods.** `save`, `findByProductId/Keyword`, `findByProductIds`

#### ScoreCachePort (Redis)
**Methods.** `cacheProductScore`, `getProductScore(s)`, `cacheKeywordScore`, `getKeywordScore`, `updateProductStats`, `getProductStats`, `updateKeywordStats`, `getKeywordStats`

## 11. Cross-Context Integration

### Consumed Events
- `analytics.event.collected` → `EventIngestionConsumer`
- 검색 클릭/노출 이벤트 → `SearchClickConsumer`, `SearchImpressionConsumer` → BanditStateRedisWriter

### Emitted Events
- `analytics.score.updated` → search BC

### Outgoing (special)
- → search BC: `BanditStateRedisWriter` 가 search 의 Bandit state를 Redis에 직접 쓰기 (**🔴 cross-BC 직접 쓰기 — 경계 침범 검토 필요**)

### Shared Terms
| 용어 | 본 BC | 다른 BC | 해결 |
|---|---|---|---|
| **Score** | raw aggregation + 가중치 합 | search BC의 Bandit posterior | **다른 개념** — 명시적 분리 |
| **ProductScore** | 본 BC 권위 | search 가 ScoredProductDocument 산정에 활용 | 본 BC가 SSOT |
| **CTR/CVR** | analytics 가 산출 | search 가 ranking signal로 활용 | 본 BC가 계산 권위 |
| **ExperimentMetric** | 본 BC 계산 | experiment BC가 GetExperimentMetricsUseCase 호출 | analytics 가 ClickHouse 쿼리 권위 |
