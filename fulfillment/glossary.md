## 1. Bounded Context Overview

Fulfillment BC는 **주문 이행(picking → packing → shipping → delivery)** 단계 관리. Order BC가 `order.order.completed` 발행 시 FulfillmentOrder를 생성하고, 상태 전이는 명시적 그래프(canTransitionTo)로 강제한다. 이행 상태별 도메인 이벤트(sealed)를 emit하여 추적·알림에 활용. Warehouse BC와 연결.

---

## 2. Aggregates & Entities

### FulfillmentOrder
**Type.** Aggregate
**Definition.** Order의 결제 완료 후 이행 단계 표현. 출고·배송 추적 단위.
**Lifecycle.** `create(orderId, warehouseId)` → PENDING → PICKING → PACKING → SHIPPED → DELIVERED (또는 임의 단계에서 CANCELLED)
**Invariants.**
- 상태 전이는 `FulfillmentStatus.canTransitionTo()` 그래프에 의해 강제 (위반 시 `InvalidFulfillmentStateException`)
- **DELIVERED · CANCELLED 는 종착 상태** (전이 불가)
- SHIPPED → CANCELLED 전이 **불가** (이미 출고되면 취소 불가, 도메인 그래프상)
- (orderId, warehouseId) 는 생성 후 불변
- `transition()` 메서드는 도메인 이벤트를 반환 — **이벤트 발행은 호출자 책임**
**Related events.** emits: `Created`, `StatusChanged`, `Shipped`, `Delivered`, `Cancelled` (FulfillmentEvent sealed)
**Avoid.** Order (commerce BC와 구분 — 본 BC는 항상 "FulfillmentOrder"), Shipment (배송 자체와 구분)
**Code.** `com.kgd.fulfillment.domain.fulfillment.model.FulfillmentOrder`
**Found in.** `fulfillment/domain/`

## 5. Domain Events

### FulfillmentEvent (sealed)
**Type.** Domain Event
**Definition.** FulfillmentOrder 상태 변경 sealed 이벤트. 5종.
**Code.** `com.kgd.fulfillment.domain.fulfillment.event.FulfillmentEvent`

| Subtype | Trigger | Payload |
|---|---|---|
| `Created` | (현재 도메인 코드에는 emit 부분 미확인 — 보통 UseCase 책임) | fulfillmentId, orderId, warehouseId |
| `StatusChanged` | `transition()` 호출, 결과가 SHIPPED/DELIVERED/CANCELLED **아닌** 경우 | fulfillmentId, orderId, from, to |
| `Shipped` | `transition(SHIPPED)` | fulfillmentId, orderId |
| `Delivered` | `transition(DELIVERED)` | fulfillmentId, orderId |
| `Cancelled` | `transition(CANCELLED)` 또는 `cancel()` | fulfillmentId, orderId |

**중요.** 상태 전이의 **특정 종착점**은 별도 sealed subtype을, **중간 전이**는 일반 `StatusChanged`로 표현. 이는 외부 소비자가 종착 상태를 명확히 구독할 수 있도록 함.

## 6. Policies & Invariants

### 상태 전이 그래프 (FulfillmentStatus.canTransitionTo)

```
PENDING  → PICKING | CANCELLED
PICKING  → PACKING | CANCELLED
PACKING  → SHIPPED | CANCELLED
SHIPPED  → DELIVERED        (CANCELLED 불가!)
DELIVERED → (no transition, terminal)
CANCELLED → (no transition, terminal)
```

| 출처 | 규칙 |
|---|---|
| `FulfillmentStatus` enum + `canTransitionTo()` | 위 그래프 강제 |
| `InvalidFulfillmentStateException` | 그래프 위반 차단 |
| 출고 후 취소 금지 | SHIPPED → CANCELLED 불가 (비즈니스 결정) |
| 종착 상태 불가역 | DELIVERED, CANCELLED |
| `FulfillmentNotFoundException` | id 조회 실패 |
