# PLANS.md

## ExecPlan Rules

### When to Use
- 2+ step work
- Long-running tasks
- High uncertainty work

### Required Components
- Purpose / Big Picture
- Plan of Work (file paths, change points explicit)
- Validation (commands and expected results)
- Progress (timestamped checklists)
- Decision Log (decisions/reasons/impacts)

### Writing Principles
- Self-contained (no external link dependency)
- Terms immediately defined
- Repeatable steps
- Failure/retry methods included
