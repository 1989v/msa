## 1. Bounded Context Overview

Wishlist BC는 **회원별 상품 위시리스트 관리**. 매우 단순 — (memberId, productId) pair에 unique 제약(JPA 레벨), 도메인 모델은 WishlistItem 하나뿐. memberId는 JWT(X-User-Id 헤더)에서 추출. product 삭제·member 탈퇴 시 Kafka 이벤트를 소비하여 자동 정리.

## 2. Aggregates & Entities

### WishlistItem
**Type.** Aggregate (얇은 Aggregate — 단일 객체)
**Definition.** 특정 Member가 관심 등록한 Product 항목. (memberId, productId) unique.
**Lifecycle.** `create(memberId, productId)` → (불변, 삭제만 가능 — 수정 메서드 없음)
**Invariants.**
- `memberId > 0` — 0/음수 ID 금지
- `productId > 0` — 0/음수 ID 금지
- (memberId, productId) **unique** — JPA constraint 레벨 (도메인 코드에는 없음)
- 생성 후 모든 필드 불변 (수정 메서드 부재)
**Related events.** — (본 BC는 emit 없음; 외부 이벤트 소비 BC)
**Avoid.** Favorite, Bookmark (의미 분화 시)
**Code.** `com.kgd.wishlist.domain.model.WishlistItem`
**Found in.** `wishlist/domain/model/`
