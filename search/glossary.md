## 1. Bounded Context Overview

Search BC는 **Elasticsearch 기반 읽기 전용 검색 모델 서비스**. CDC + Kafka 로 product BC 변경을 비동기 인덱싱. 검색 결과는 ES native score + Bandit posterior 의 hybrid reranking 으로 최종 랭킹. multi-armed bandit (Thompson sampling) 으로 노출·클릭 피드백 기반 적응 학습. 무중단 리인덱스는 alias swap.

## 2. Aggregates & Entities

### ProductDocument
**Type.** Aggregate (data class — 검색 인덱스의 비정규화 표현)
**Definition.** Product 의 검색 인덱스 표현. product BC 의 권위 모델을 CDC/Kafka 로 동기화한 **읽기 모델**. ES document 단위로 인덱스됨.
**Lifecycle.** `product.item.created` 수신 시 생성 → `product.item.updated` 수신 시 갱신 → `analytics.score.updated` 수신 시 score 필드 갱신
**Invariants.**
- ES 측 권위 (이중 권위 아님 — product BC가 카탈로그 SSOT)
- `popularityScore / ctr / cvr / scoreUpdatedAt` 는 analytics BC 에서 주입된 외부 score 필드
- `categoryId` 는 nullable — 미분류 상품 허용
- `status: String` (enum 아님) — product BC의 ProductStatus 와 다른 표현 (Flagged: enum 동기화 검토)
**Avoid.** Product (product BC의 권위 모델과 혼동 금지), ProductIndex (port 이름과 충돌)
**Code.** `com.kgd.search.domain.product.model.ProductDocument`

### BanditState
**Type.** Aggregate (data class)
**Definition.** 특정 BanditKey 의 현재 학습 상태. 누적 노출·클릭 수와 마지막 갱신 시각.
**Lifecycle.** `empty(key)` 또는 신규 ImpressionEvent 수신 시 생성 → 노출/클릭 누적 → time decay 가능
**Invariants.**
- `clicks >= 0` — 음수 클릭 금지
- **`impressions >= clicks`** — 클릭이 노출보다 많을 수 없음 (논리적 invariant)
- `ageDays(now)` — lastUpdatedAt 기준 경과일 (time decay 가중치 산정에 활용)
**Code.** `com.kgd.search.domain.bandit.model.BanditState`

## 3. Value Objects

### ScoredProductDocument
**Type.** Value Object (data class)
**Definition.** ProductDocument + ES native score(`esScore`). reranker 입력.
**Code.** `com.kgd.search.domain.product.model.ScoredProductDocument`

### BanditKey
**Type.** Value Object (data class)
**Definition.** Bandit 학습 단위 식별자 — `(categoryId, productId)` 결합.
**Invariants.**
- `categoryId.isNotBlank() ?: DEFAULT_CATEGORY` ("_default_") — 빈 카테고리는 기본값으로 폴백
- `redisField()` → "categoryId:productId" 문자열 (Redis key 표현)
**Code.** `com.kgd.search.domain.bandit.model.BanditKey`

### BanditPosterior
**Type.** Value Object (data class — Beta 분포 파라미터)
**Definition.** 베이지안 사후 분포. Beta(α, β) — α=클릭, β=노출-클릭 (사전분포 + 관측 누적).
**Invariants.**
- `alpha > 0.0`, `beta > 0.0` — Beta 분포 파라미터 양수 강제
- `mean() = alpha / (alpha + beta)` — 분포 평균
**Code.** `com.kgd.search.domain.bandit.model.BanditPosterior`

### ImpressionEvent / ClickEvent
**Type.** Value Object (data class)
**Definition.** 검색 결과 노출·클릭 단일 이벤트.
**Payload.** `searchId + key + position + userId?(nullable) + occurredAt`
**Code.** `com.kgd.search.domain.bandit.model.{ImpressionEvent, ClickEvent}`

**Type.** Domain Service (app 레이어 분류 예외)
**Definition.** BanditPosterior 에서 표본을 추출 → 가장 큰 표본의 팔(arm)을 선택하는 reranker. 검색 결과 ScoredProductDocument 리스트를 받아 재정렬.
**Code.** `com.kgd.search.app.bandit.ThompsonReranker` + `BetaSampler`

## 5. Domain Events

### ImpressionEvent / ClickEvent
**Type.** Domain Event (Bandit 학습 신호)
**Trigger.** 검색 결과 노출·사용자 클릭 시
**Code.** `com.kgd.search.domain.bandit.model.{ImpressionEvent, ClickEvent}`
**Consumed by.** `BanditEventPort` → analytics BC 가 BanditStateRedisWriter 로 상태 갱신

### Consumed Events (외부에서 본 BC로)
- **product BC**: `product.item.created`, `product.item.updated` (Kafka, consumer group `search-indexer`) → ProductIndexingConsumer → ProductDocument 갱신
- **analytics BC**: `analytics.score.updated` (Kafka) → ProductScoreUpdateConsumer → ProductDocument 의 score 필드 갱신

## 6. Policies & Invariants

| 출처 | 규칙 |
|---|---|
| `BanditKey.DEFAULT_CATEGORY` | "_default_" — 카테고리 미분류 시 폴백 |
| `BanditPosterior.init` | alpha, beta > 0 (Beta 분포 정의역) |
| `BanditState.init` | clicks ≥ 0, **impressions ≥ clicks** |
| 읽기 전용 모델 | ES 는 product DB의 read model — 직접 쓰기 금지 (CLAUDE.md) |
| Kafka 소비 토픽 | `product.item.created`, `product.item.updated` (consumer group: `search-indexer`) |
| Batch 리인덱싱 | alias swap 방식 — 무중단 전환 (CLAUDE.md) |
| 멱등성 | ADR-0012 — 중복 이벤트 방어 필수 |
| `searchScored` default | `esScore = 0.0` wrap — 어댑터가 반드시 override 해야 의미 있음 (KDoc) |

## 7. Ports

### Outbound

#### ProductSearchPort
**Definition.** ES 검색 추상화. `search(keyword, Pageable) → Page<ProductDocument>` + `searchScored` (esScore 동봉, reranker 입력용).
**KDoc 인용.** "ES `_score` 를 동봉해 반환. application 레이어의 reranker (e.g. ThompsonReranker) 가 esScore 를 hybrid 계산에 사용한다."
**Adapter.** `ProductSearchAdapter` (Elasticsearch RestClient)

#### ProductIndexPort
**Definition.** ES 인덱싱 추상화 — `indexProduct(doc)`, `bulkIndex(docs)`.
**Adapter.** `ProductSearchAdapter` 또는 별도 indexing adapter

#### BanditStatePort
**Definition.** BanditState 조회 (Redis). `fetch(key)`, `fetchBatch(keys)`.
**Adapter.** `BanditStateRedisAdapter`

#### BanditEventPort
**Definition.** Bandit 학습 이벤트 발행. `recordImpression(event)`, `recordClick(event)`.
**Adapter.** `BanditEventKafkaAdapter`

## 11. Cross-Context Integration

### Consumed Events
- product BC → `product.item.created` / `product.item.updated` → ProductIndexingConsumer
- analytics BC → `analytics.score.updated` → ProductScoreUpdateConsumer

### Emitted Events
- `ImpressionEvent`, `ClickEvent` (via BanditEventPort/Kafka)

### Special Integration
- **analytics BC가 search BC의 Redis state 에 직접 쓰기** (BanditStateRedisWriter) — 본 BC의 BanditStatePort 가 읽기만 노출하는 이유. **🔴 경계 침범 검토 필요**

### Shared Terms
| 용어 | 본 BC | 다른 BC | 해결 |
|---|---|---|---|
| **Product** | ProductDocument (읽기 모델) | product BC: 권위 모델 | product가 SSOT, 본 BC는 비정규화 사본 |
| **Score** | ES `_score` + bandit posterior | analytics BC의 ProductScore | **다른 개념** — 본 BC는 ranking signal, analytics는 raw aggregation |
| **BanditKey** | 본 BC 전용 | experiment BC의 Variant 와 다른 패러다임 | Bandit=연속학습, Experiment=사전할당 |
| **Variant / Arm** | bandit 용어로 "arm" | experiment BC의 Variant 와 다름 | 본 BC는 "arm" 단어 사용 자제 |
