## 1. Bounded Context Overview

Member BC는 **회원 식별·프로필 관리**. **최소 개인정보 원칙** (email, name, SSO 제공자/ID만 저장 — 주소·전화번호 등 추가 PII는 본 BC가 다루지 않음). Auth BC가 OAuth 로그인 시 `/api/members/sso` 를 호출하여 회원을 조회·생성. 탈퇴 시 `member.withdrawn` Kafka 이벤트 발행(향후), wishlist BC가 소비하여 데이터 정리.

## 2. Aggregates & Entities

### Member
**Type.** Aggregate
**Definition.** 서비스에 가입한 고객 계정. email + (ssoProvider, ssoProviderId) 결합으로 식별. 이름·상태만 변경 가능, 그 외 필드는 불변.
**Lifecycle.** `create(email, name, ssoProvider, ssoProviderId)` → ACTIVE → {`updateName()` 반복 | `suspend()` → SUSPENDED ↔ `activate()` | `withdraw()` → WITHDRAWN}
**Invariants.**
- `email.isNotBlank()`, `name.isNotBlank()` (생성·updateName 시)
- `withdraw()` 는 ACTIVE 상태에서만 가능 (이미 탈퇴/정지된 회원 재탈퇴 금지)
- `suspend()` 는 ACTIVE 상태에서만 가능
- `activate()` 는 SUSPENDED 상태에서만 가능 — **WITHDRAWN 은 활성화 불가** (탈퇴는 불가역)
- email + ssoProvider + ssoProviderId 는 생성 후 불변
**Related events.** emits: `member.withdrawn` (Kafka, **향후 — 현재 미구현**)
**Avoid.** User, Customer, Account (의미 분화 시), gifticon BC의 ShareMember(부분 프로젝션)와 혼동 금지
**Code.** `com.kgd.member.domain.model.Member`
**Found in.** `member/domain/model/`
**References.** CLAUDE.md의 "최소 개인정보" 원칙

## 6. Policies & Invariants

| 출처 | 규칙 |
|---|---|
| `MemberStatus` enum | `ACTIVE / SUSPENDED / WITHDRAWN` 3상태 |

| 최소 개인정보 원칙 | email + name + SSO 정보만 저장 (CLAUDE.md) |
