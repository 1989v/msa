## 1. Bounded Context Overview

Experiment BC는 **A/B 테스트 실험 관리·버킷 할당·결과 분석**. MySQL 단독 소유. 결과 분석은 ClickHouse 직접 접근 금지 — analytics BC API 호출. 버킷 할당은 common 모듈 BucketAssigner(MurmurHash3 결정적 해싱). 상태 전이는 명시적 그래프(canTransitionTo).

## 2. Aggregates & Entities

### Experiment
**Type.** Aggregate (data class with init)
**Definition.** A/B 테스트 단위. name + variants + trafficPercentage + status 보유.
**Lifecycle.** `create(...)` → DRAFT → RUNNING → {PAUSED ↔ RUNNING | COMPLETED}; COMPLETED는 종착
**Invariants.**
- `trafficPercentage in 0..100` (init 시) — 음수/100 초과 금지
- `variants.isNotEmpty()` (init 시) — 최소 1개 변형
- **variants 가중치 합 == 100** (`variants.sumOf { it.weight } == 100`) — 트래픽 분배 합 100% 강제
- 상태 전이는 `ExperimentStatus.canTransitionTo()` 그래프에 따름 (위반 시 `BusinessException INVALID_INPUT`)
- COMPLETED는 종착 상태 — 어떤 상태로도 전이 불가
**Related events.** — (현재 도메인 이벤트 없음)
**Avoid.** Test (단위 테스트와 혼동), ABTest (full word 사용)
**Code.** `com.kgd.experiment.domain.model.Experiment`

### Variant
**Type.** Entity (Experiment 내부)
**Definition.** Experiment 의 대안 그룹 (Control / Treatment). name + weight + config map.
**Invariants.**
- `weight > 0` — 0 또는 음수 가중치 금지
- `name.isNotBlank()`
- (가중치 합 == 100 검증은 Experiment 레벨)
**Avoid.** Arm (multi-armed bandit 용어와 구분 — search BC와 다름), Bucket (할당 결과와 구분)
**Code.** `com.kgd.experiment.domain.model.Variant`

## 4. Domain Services

### StatisticalSignificance (object)
**Type.** Domain Service
**Definition.** 통계적 유의성 검정 도메인 로직. 현재 Two-proportion Z-test 구현.
**Code.** `com.kgd.experiment.domain.model.StatisticalSignificance`

**Method.**

#### `twoProportionZTest(controlSuccess, controlTotal, treatmentSuccess, treatmentTotal, significanceLevel=0.05): SignificanceResult`
**Algorithm.**
```
p1 = controlSuccess / controlTotal       (control 전환률)
p2 = treatmentSuccess / treatmentTotal   (treatment 전환률)
pPooled = (s1+s2) / (n1+n2)              (pooled proportion)
se = sqrt(pPooled*(1-pPooled) * (1/n1 + 1/n2))   (standard error)
z = (p1 - p2) / se
pValue = 2 * normalCdf(-|z|)             (two-tailed)
isSignificant = pValue < significanceLevel
confidenceLevel = 1 - pValue
```
**Edge cases.** 양 그룹 total == 0 또는 se == 0 → not significant (안전한 기본값)
**Internal.** `normalCdf` 는 Abramowitz-Stegun 근사 사용 (z ∈ [-8, 8] 범위 외는 0/1 클리핑)

## 6. Policies & Invariants

### ExperimentStatus 전이 그래프 (canTransitionTo)

```
DRAFT     → RUNNING
RUNNING   → PAUSED | COMPLETED
PAUSED    → RUNNING | COMPLETED
COMPLETED → (no transition, terminal)
```

| 출처 | 규칙 |
|---|---|
| `ExperimentStatus` enum | DRAFT/RUNNING/PAUSED/COMPLETED |
| `Experiment.init` require | `trafficPercentage 0..100`, `variants ≥ 1개`, **weight 합 100** |
| `Variant.init` require | `weight > 0`, `name 비어있지 않음` |
| `Experiment.changeStatus` | 그래프 위반 시 `BusinessException INVALID_INPUT` |
| `BucketAssigner` (common) | MurmurHash3 결정적 해싱 (CLAUDE.md) |
| `StatisticalSignificance` | Two-proportion Z-test, significance level 0.05 기본 |
