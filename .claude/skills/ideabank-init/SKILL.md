# Idea Bank — Init (1차 PRD 생성)

tempidea.md에 기록된 날 것의 아이디어를 읽고, 구조화된 1차 PRD 초안을 `ideabank/docs/` 하위에 생성한다.

## 트리거

- `/ideabank-init` — PRD가 아직 없는 아이디어 목록을 보여주고 선택
- `/ideabank-init 3` — 3번 아이디어의 PRD를 바로 생성
- 자연어: "아이디어 정리해줘", "tempidea PRD 만들어줘", "아이디어 초기화", "아이디어 구체화 시작"

## 실행 단계

### 1. 현황 파악

1. `ideabank/tempidea.md` 파일을 읽는다 (Read 도구)
2. `ideabank/docs/` 디렉토리를 스캔하여 이미 생성된 PRD 목록을 확인한다 (Glob 도구)
3. Pending Ideas 섹션에서 아직 PRD가 없는 아이디어를 식별한다

### 2. 아이디어 선택

- **번호가 주어진 경우**: 해당 번호의 아이디어를 바로 처리
- **번호가 없는 경우**: PRD 미생성 아이디어 목록을 표 형태로 출력하고 사용자에게 선택을 요청

출력 예시:
```
| # | 아이디어 | PRD 상태 |
|---|---------|---------|
| 1 | 퀀트 트레이더 자동매매 | 미생성 |
| 3 | IT개념 코드 위치 추적 서비스 | 미생성 |
| 6 | 한동 파이프 웹 서비스 | 미생성 |
```

### 3. PRD 초안 생성

선택된 아이디어의 tempidea.md 내용(본문 + 하위 항목)을 기반으로 1차 PRD를 작성한다.

**파일 경로**: `ideabank/docs/{번호}-{slug}.md`
- slug는 아이디어 핵심 키워드의 영문 kebab-case (예: `1-quant-trader.md`, `3-code-dictionary.md`)

**PRD 템플릿**:

```markdown
---
id: {번호}
title: {한글 제목}
status: draft
created: {YYYY-MM-DD}
updated: {YYYY-MM-DD}
service-dir: {예상 서비스 디렉토리명, 미정이면 TBD}
---

# {한글 제목}

## 1. 개요
<!-- 이 서비스가 무엇이고 왜 필요한지 1-2문단 -->

## 2. 핵심 기능
<!-- 주요 기능을 bullet으로 나열 -->

## 3. 사용자 시나리오
<!-- 주요 유저 플로우 2-3개 -->

## 4. 기술 스택 (예상)
<!-- 언어, 프레임워크, 인프라 등 -->

## 5. 서비스 구조 (예상)
<!-- 대략적인 모듈/컴포넌트 구조 -->

## 6. 미결 사항 / 질문
<!-- 브레인스토밍에서 논의가 필요한 항목 -->

## 7. 원본 아이디어
<!-- tempidea.md에서 가져온 원문 그대로 보존 -->
```

**작성 원칙**:
- tempidea.md의 원본 내용을 `7. 원본 아이디어` 섹션에 그대로 보존
- 섹션 1~5는 원본 내용을 기반으로 AI가 구조화하여 작성
- 섹션 6은 원본에서 `?`로 끝나거나 불확실한 항목, 추가 논의가 필요한 사항을 정리
- 과도한 추측이나 없는 내용 창작 금지 — 원본에 근거한 정리만 수행

### 4. tempidea.md PRD 현황 표 업데이트

PRD 생성 후, `ideabank/tempidea.md`의 **PRD 현황** 표에 해당 아이디어의 행을 추가/업데이트한다 (Edit 도구).

```markdown
## PRD 현황

| # | 제목 | PRD | 상태 |
|---|------|-----|------|
| {번호} | {한글 제목} | [{번호}-{slug}.md](docs/{번호}-{slug}.md) | draft |
```

- 이미 해당 번호의 행이 있으면 업데이트
- 없으면 번호 순서에 맞게 행 추가
- PRD 현황 표 자체가 없으면 `## Pending Ideas` 위에 생성

### 5. 결과 보고

PRD 생성 후 사용자에게:
- 생성된 PRD 파일 경로
- PRD 요약 (핵심 기능 3줄 이내)
- 다음 단계 안내: `/ideabank-bs {번호}` 로 브레인스토밍 진행 가능

### 6. 다수 아이디어 일괄 처리

`/ideabank-init all` 호출 시:
- Pending Ideas 중 PRD 미생성인 모든 아이디어에 대해 순차적으로 PRD 생성
- 각 PRD 생성 후 PRD 현황 표에 행 추가
- 완료 후 전체 현황 표 출력

## 에러 처리

- `ideabank/tempidea.md`가 없는 경우: 파일 생성 안내
- `ideabank/docs/` 디렉토리가 없는 경우: 자동 생성
- 이미 해당 번호의 PRD가 존재하는 경우: 덮어쓰기 여부 확인
- 존재하지 않는 번호 지정 시: 사용 가능한 번호 목록 출력
