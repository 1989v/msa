# Idea Bank — Brainstorm (PRD 브레인스토밍)

이미 생성된 PRD를 사용자와 대화형으로 검토하고 가다듬는다. 핑퐁 방식의 질의응답으로 PRD의 완성도를 높인다.

## 트리거

- `/ideabank-bs` — draft 상태인 PRD 목록을 보여주고 선택
- `/ideabank-bs 3` — 3번 PRD를 바로 브레인스토밍 시작
- 자연어: "아이디어 브레인스토밍 하자", "PRD 가다듬자", "아이디어 같이 논의하자", "아이디어 보완하자"

## 실행 단계

### 1. PRD 선택

1. `ideabank/docs/` 디렉토리를 스캔하여 PRD 목록을 확인 (Glob 도구)
2. 각 PRD의 frontmatter에서 status를 읽어 상태 확인 (Read 도구)

- **번호가 주어진 경우**: 해당 PRD를 바로 열기
- **번호가 없는 경우**: PRD 목록을 상태와 함께 표시하고 선택 요청

출력 예시:
```
| # | 제목 | 상태 | 최종 수정 |
|---|------|------|----------|
| 1 | 퀀트 트레이더 자동매매 | draft | 2026-03-21 |
| 3 | IT개념 코드 위치 추적 서비스 | refined | 2026-03-20 |
```

### 2. PRD 분석 및 대화 시작

선택된 PRD를 읽고, 사용자에게 다음을 제시:

1. **현재 PRD 요약** — 핵심 기능과 구조를 3-5줄로 요약
2. **미결 사항** — 섹션 6의 미결 사항/질문 목록
3. **첫 번째 질문** — 가장 중요한 미결 사항이나 설계 방향에 대해 **한 가지** 질문

### 3. 핑퐁 브레인스토밍

대화 원칙:
- **한 번에 한 가지 질문만** — 여러 질문을 한꺼번에 던지지 않는다
- **가능하면 선택지 제공** — "A vs B 중 어느 방향?" 형태로 구체적 선택지 제시
- **사용자 답변을 즉시 PRD에 반영** — 답변 받을 때마다 해당 섹션 업데이트
- **근거 있는 추천** — 기술적 의견을 줄 때는 이유와 트레이드오프 설명
- **사용자가 "됐어", "이정도면 됐다", "완료" 등 종료 의사를 밝히면** 브레인스토밍 종료

논의 진행 흐름:
1. 미결 사항 해소 (섹션 6 항목들)
2. 핵심 기능 우선순위 및 MVP 범위
3. 기술 스택 확정
4. 서비스 구조 구체화
5. 추가 아이디어 또는 개선점

### 4. PRD 업데이트

브레인스토밍 중/후에 PRD 파일을 업데이트:

- 논의 결과를 해당 섹션에 반영 (Edit 도구)
- frontmatter의 `updated` 날짜 갱신
- 충분히 가다듬어졌으면 `status: draft` → `status: refined` 변경
- 사용자가 구현 준비 완료를 선언하면 `status: refined` → `status: ready` 변경
- **상태 변경 시 `ideabank/tempidea.md`의 PRD 현황 표도 동기화** (해당 행의 상태 컬럼 업데이트)

### 5. 브레인스토밍 종료

종료 시 사용자에게:
- 변경된 내용 요약 (이번 세션에서 결정된 사항)
- 현재 PRD 상태 (draft / refined / ready)
- 다음 단계 안내:
  - `refined`: "추가 브레인스토밍이 필요하면 `/ideabank-bs {번호}` 재호출"
  - `ready`: "`/ideabank-impl {번호}` 로 구현 시작 가능"

## PRD 상태 흐름

```
draft → refined → ready → implemented
 (init)   (bs)      (bs)     (impl)
```

- **draft**: init으로 1차 생성된 상태
- **refined**: 1회 이상 브레인스토밍으로 보완된 상태
- **ready**: 구현 가능한 수준으로 확정된 상태
- **implemented**: 서비스로 구현 완료

## 대화 스타일

- 딱딱한 문서 리뷰가 아닌 **자연스러운 대화** 톤
- 사용자의 아이디어에 공감하면서도 **현실적 제약**을 짚어주기
- "이건 좋은데, 이런 문제가 있을 수 있어요" 식의 건설적 피드백
- 기술적으로 더 나은 대안이 있으면 적극 제안

## 에러 처리

- PRD가 없는 번호 지정 시: `/ideabank-init {번호}` 로 먼저 생성하라고 안내
- `ideabank/docs/` 가 비어있는 경우: `/ideabank-init` 먼저 실행 안내
