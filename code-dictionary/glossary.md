## 1. Bounded Context Overview

Code-Dictionary BC는 **코드베이스 어휘·서비스 카탈로그·개념 그래프** 관리. 메타 레벨 서비스 — 다른 BC들이 사용하는 개념(Concept)을 등재하고 코드 위치(CodeLocation)와 연결한다. OpenSearch로 검색·자동완성·관계 그래프 제공.

---

## 2. Aggregates & Entities

### Concept
**Type.** Aggregate
**Definition.** 코드베이스의 단일 개념. conceptId 문자열로 식별. 이름·카테고리·난이도·설명·동의어·관련 개념 보유.
**Lifecycle.** `create(...)` → 영속 → `update(name?, category?, level?, description?)` / `updateSynonyms()` / `updateDescription()` 반복
**Invariants.**
- `conceptId.isNotBlank()`, `name.isNotBlank()` (생성·update 시)
- `synonyms`, `relatedConceptIds` 는 List<String> — 빈 리스트 허용
- 생성 시 ConceptCategory + ConceptLevel 필수
**Avoid.** Term (Term은 본 BC 전체 영역의 핵심 어휘 — 별도 의미 분리 필요), `glossary.md`의 Term과 혼동 금지
**Code.** `com.kgd.codedictionary.domain.concept.model.Concept`

### ConceptIndex
**Type.** Aggregate
**Definition.** Concept을 코드 위치(CodeLocation)에 연결하는 색인 단위. 같은 Concept이 여러 위치에 등장 가능.
**Lifecycle.** `create(conceptId, location, snippet?, description?, gitCommitHash?)` → indexedAt 자동 → 영속
**Invariants.**
- `conceptId.isNotBlank()` (생성 시)
- `indexedAt = LocalDateTime.now()` 자동 (restore 시는 보존)
- gitCommitHash 선택 (Git 추적 가능 시)
**Code.** `com.kgd.codedictionary.domain.index.model.ConceptIndex`

### Service
**Type.** Aggregate (서비스 카탈로그)
**Definition.** MSA 의 단일 서비스 카탈로그 표현. code(식별자) + name + port + isPrivate + 표시 순서 + 보유 Concept 리스트.
**Lifecycle.** `create(...)` → 영속, 필드 변경 가능 (mutable properties)
**Invariants.**
- `code.isNotBlank()`, `name.isNotBlank()` (생성 시)
- `port: Int?` (null 허용 — UI 서비스 등)
- `isPrivate` flag (내부 전용 표시)
**Avoid.** Spring `@Service` annotation, settings.gradle.kts 의 module 과 혼동 금지 — 본 BC의 Service는 카탈로그 메타데이터
**Code.** `com.kgd.codedictionary.domain.service.model.Service`

## 3. Value Objects

### CodeLocation
**Type.** Value Object (data class with init)
**Definition.** 코드의 단일 위치. filePath + lineStart + lineEnd + 선택적 gitUrl.
**Invariants.**
- `filePath.isNotBlank()` (init 시)
- `lineStart > 0`
- `lineEnd >= lineStart` — 역순 라인 금지
- gitUrl 은 선택적 (외부 링크 가능 시)
**Code.** `com.kgd.codedictionary.domain.index.model.CodeLocation`

## 6. Policies & Invariants

**Type.** Policy enum
**Values.** `BASICS / DATA_STRUCTURE / ALGORITHM / DESIGN_PATTERN / CONCURRENCY / DISTRIBUTED_SYSTEM / ARCHITECTURE / INFRASTRUCTURE / DATA / SECURITY / NETWORK / TESTING / LANGUAGE_FEATURE`

### ConceptLevel (3단계)
**Type.** Policy enum
**Values.** `BEGINNER / INTERMEDIATE / ADVANCED`

| 출처 | 규칙 |
|---|---|
