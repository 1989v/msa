## 1. Bounded Context Overview

Warehouse BC는 **물리 창고 마스터 데이터** 관리. 이름·주소·위경도·활성 상태를 보유. 현재 단순 CRUD + active toggle. inventory/fulfillment BC가 warehouseId로 참조한다.

## 2. Aggregates & Entities

### Warehouse
**Type.** Aggregate
**Definition.** 물리적 재고 보관 위치. 이름·주소·좌표(위경도)·활성 상태를 보유.
**Lifecycle.** `create(name, address, lat, long)` → active=true → `deactivate()` ↔ `activate()` 토글
**Invariants.**
- `name.isNotBlank()` (생성 시)
- `deactivate()` 는 active 상태에서만 가능 (이중 비활성화 차단)
- `activate()` 는 inactive 상태에서만 가능 (이중 활성화 차단)
- 좌표(latitude, longitude)는 Double primitive — **범위 검증 없음** (Flagged)
- 주소(address) — **빈 문자열 허용** (Flagged: 의도된 설계인지)
**Related events.** — (현재 도메인 이벤트 없음)
**Avoid.** Location, Storage (단순 좌표·저장 의미와 구분)
**Code.** `com.kgd.warehouse.domain.warehouse.model.Warehouse`
