## 1. Bounded Context Overview

Inventory BC는 **재고의 SSOT** (ADR-0013) — Product BC는 카탈로그 SSOT, 재고는 본 BC가 권위. 가용 재고(`availableQty`)와 예약 재고(`reservedQty`)를 분리 관리하여 가예약(Reservation) 모델을 지원한다. Order BC가 결제 전 단계에서 가예약을 잡고, 결제 완료 시 확정(confirm)·실패 시 해제(release)·시간 초과 시 만료(expire) 처리한다. Outbox 패턴(ADR-0012 멱등 consumer 결합)으로 트랜잭션 일관성을 보장하며 ClickHouse/Redis로 캐시·메트릭을 분리한다.

## 2. Aggregates & Entities

### Inventory
**Type.** Aggregate
**Definition.** 특정 (Product, Warehouse) 조합의 재고 권위 보유체. 가용수량과 예약수량을 분리 보유하여 동시 예약 경합을 차단한다.
**Lifecycle.** `create(initialQty≥0)` → `reserve / release / confirm / receive` 반복 → (논리적 종료 없음, 영속)
**Invariants.**
- `initialQty >= 0` (생성 시)
- `availableQty >= qty` (reserve 시) — 위반 시 `InsufficientStockException`
- `reservedQty >= qty` (release / confirm 시)
- `qty > 0` (receive 시)
- `version` 필드로 **optimistic concurrency control** (Spring Data JPA `@Version`)
- 총 보유량 = `availableQty + reservedQty` 단조 보존 (reserve/release는 두 필드 간 이동, receive만 합 증가)
**Related events.** emits: `StockReserved`, `StockReleased`, `StockConfirmed`, `StockReceived` (InventoryEvent sealed)
**Avoid.** Stock (단순 수량 의미와 구분 — Inventory는 (productId, warehouseId) 결합 단위)
**Code.** `com.kgd.inventory.domain.inventory.model.Inventory`
**Found in.** `inventory/domain/`, ADR-0013
**References.** ADR-0013 (Product-Inventory SSOT), ADR-0011 (inventory-fulfillment service)

### Reservation
**Type.** Aggregate
**Definition.** Order 확정 전 단계의 가예약. 특정 Order가 특정 Inventory의 일부를 일시 보유. TTL(기본 30분) 후 만료 가능.
**Lifecycle.** `create(ttlMinutes=30)` → ACTIVE → {`confirm()` → CONFIRMED | `cancel()` → CANCELLED | `expire()` → EXPIRED}
**Invariants.**
- 상태 전이는 **ACTIVE에서만** 가능 (CONFIRMED/CANCELLED/EXPIRED에서 전이 시도 시 `InvalidReservationStateException`)
- `expire()`는 `expiredAt` 시점이 지나야만 호출 가능 (위반 시 `IllegalStateException`)
- (orderId, productId, warehouseId) 필드는 생성 후 불변
- ttlMinutes 기본 30분 (인자로 override 가능)
**Related events.** emits: `Expired` (ReservationEvent sealed) — confirm/cancel은 현재 명시적 도메인 이벤트 없음 (TBD: 추가 필요한지 grilling)
**Avoid.** Hold, Lock (가예약 의미와 다름), 외부 mrt-package의 "Reservation"(여행 예약)과 단어 충돌 — 본 BC는 상품 재고 가예약 한정
**Code.** `com.kgd.inventory.domain.reservation.model.Reservation`
**Found in.** `inventory/domain/reservation/`
**References.** ADR-0011

## 5. Domain Events

### InventoryEvent (sealed)
**Type.** Domain Event
**Definition.** Inventory Aggregate 상태 변경을 외부에 알리는 sealed 이벤트.
**Code.** `com.kgd.inventory.domain.inventory.event.InventoryEvent`
**Subtypes & emit trigger:**
| Subtype | Trigger | Payload |
|---|---|---|
| `StockReserved` | `Inventory.reserve(qty)` 성공 | productId, warehouseId, qty, orderId, availableQty |
| `StockReleased` | `Inventory.release(qty)` 성공 | 동일 |
| `StockConfirmed` | `Inventory.confirm(qty)` 성공 | 동일 |
| `StockReceived` | `Inventory.receive(qty)` 성공 | productId, warehouseId, qty, availableQty |

**Kafka 발행.** `InventoryKafkaEvents` → Outbox 경유 → Kafka 토픽 (TBD: 정확한 토픽명 grilling)

### ReservationEvent (sealed)
**Type.** Domain Event
**Definition.** Reservation 상태 변경 알림. 현재 `Expired`만 정의됨.
**Code.** `com.kgd.inventory.domain.reservation.event.ReservationEvent`
**Subtypes & emit trigger:**
| Subtype | Trigger | Payload |
|---|---|---|
| `Expired` | `ExpireReservationsUseCase` → `Reservation.expire()` | reservationId, orderId, productId, warehouseId, qty |

**Flagged.** Confirm·Cancel 이벤트 없음 — 의도된 설계인지 grilling 필요 (사가 보상 트랜잭션 시 필요할 수 있음)

## 6. Policies & Invariants

추출된 정책 규칙 (sealed/enum/Exception 기반):

| 출처 | 규칙 | 정책 의미 |
|---|---|---|
| `ReservationStatus` enum | `ACTIVE / CONFIRMED / CANCELLED / EXPIRED` 4상태 | 가예약 라이프사이클은 4상태 한정 |
| `InvalidReservationStateException` | ACTIVE 외 상태에서 confirm/cancel/expire 호출 차단 | **불가역 전이**: 종료 상태는 다시 ACTIVE로 못 돌아옴 |
| `Reservation.expire()` 내부 if | `expiredAt > now()` 이면 expire 불가 | 만료 시점 도달 전 강제 만료 금지 |
| `InsufficientStockException` | `availableQty < qty` 시 발생 | **음수 재고 금지** (overselling 차단) |
| `Inventory.create()` require | `initialQty >= 0` | 음수 초기 재고 금지 |
| `Inventory.receive()` require | `qty > 0` | 0·음수 입고 금지 |
| `Inventory.release/confirm()` require | `reservedQty >= qty` | 예약량 초과 해제·확정 금지 |
| `Inventory.version` 필드 | optimistic locking | 동시 update 충돌 시 한쪽 실패 |
| `Reservation.create()` ttlMinutes 기본 | 30분 | 가예약 TTL 정책 (조정 시 ADR 권장) |

## 8. Use Cases (Verb + Noun 사전)

| Use Case | Verb | Noun | Side effects |
|---|---|---|---|
| `ReserveStockUseCase` | **Reserve** | Stock | Inventory.reserve + Reservation 생성 + StockReserved emit |
| `ConfirmStockUseCase` | **Confirm** | Stock | Reservation.confirm + Inventory.confirm + StockConfirmed emit |
| `ConfirmStockByOrderUseCase` | Confirm | Stock (by Order) | 여러 Reservation 일괄 확정 (orderId로 lookup) |
| `ReleaseStockUseCase` | **Release** | Stock | Reservation.cancel + Inventory.release + StockReleased emit |
| `ReleaseStockByOrderUseCase` | Release | Stock (by Order) | 여러 Reservation 일괄 해제 |
| `ReceiveStockUseCase` | **Receive** | Stock | Inventory.receive + StockReceived emit (입고) |
| `ExpireReservationsUseCase` | **Expire** | Reservations | 만료 예약 일괄 expire + Expired emit |
| `GetInventoryUseCase` | Get | Inventory | 조회 (캐시 우선) |

**핵심 동사 어휘.** Reserve / Confirm / Release / Receive / Expire / Get (Cancel은 명시적 UseCase 없음 — Release로 처리)
**핵심 명사 어휘.** Stock (외부 인터페이스 명사), Inventory / Reservation (내부 Aggregate)

> **명명 주의**: 외부 API는 "Stock"으로 노출, 내부 도메인은 "Inventory" + "Reservation"으로 분리. 두 용어가 같은 개념이 아님을 명확히 — Stock은 가용량 관점의 표현일 뿐.

### Shared Terms (다른 BC에 등장하지만 의미 분리 필요)
| 용어 | 본 BC 의미 | 다른 BC | 해결 |
|---|---|---|---|
| **Stock** | 외부 API 명사 (Inventory 의 표면) | product BC가 "재고 캐시" 의미로 사용 | product는 캐시·동기화 대상, 본 BC가 SSOT |
| **Reservation** | 상품 재고 가예약 | mrt-package (별 프로젝트): 여행 예약 | 다른 시스템 — 외부 통신 시 별도 명명 |

| **Confirm / Cancel** | Reservation 도메인 메서드 | order BC: Order의 상태 전이 | 같은 단어, 다른 객체 — 컨텍스트 명시 필수 |
