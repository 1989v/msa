# 여행 OTA 추천 엔진 카탈로그 — `engines/` 디렉토리 산업 사례 정리

> 출처: 산업 사례 (여행 OTA 의 추천 엔진 모음 디렉토리 — 38개 엔진 production 코드 카탈로그)
> 작성일: 2026-05-12
> 대상: study/temp.md #20 (추천 모델링 알고리즘) 의 입력 자료

**용어 풀이 (study/ 약어 규칙)**:
- **OTA — Online Travel Agency (온라인 여행사)**: 호텔/투어/항공/렌터카 등 여행 상품을 통합 판매하는 플랫폼.
- **CF — Collaborative Filtering (협업 필터링)**: 사용자-아이템 상호작용 (조회/검색/구매 등) 기반으로 아이템 간 유사도/사용자 간 유사도를 학습.
- **CTR — Click-Through Rate (클릭률)**: 노출 대비 클릭 비율. 추천 랭킹의 핵심 시그널.
- **NER — Named Entity Recognition (개체명 인식)**: 텍스트에서 지역/카테고리/상품명 등 의미 단위 추출.
- **TNA — Tour & Activity (투어 & 액티비티)**: OTA 도메인 일반 용어, 투어 상품 카테고리.
- **NLP — Natural Language Processing (자연어 처리)**.
- **DAG — Directed Acyclic Graph (방향 비순환 그래프)**: Airflow 워크플로 단위.
- **MLM — Masked Language Model (마스크 언어 모델)**: BERT 계열 사전학습 방식.
- **gid — Group ID**: 숙소/상품 그룹 식별자.
- **DLRM — Deep Learning Recommendation Model (딥러닝 추천 모델, Meta)**.
- **MoreLikeThis (ES)**: Elasticsearch 의 텀-빈도 기반 유사 문서 검색 쿼리.
- **xsell — Cross-sell (교차 판매)**.
- **NLP 임베딩 (Sentence-BERT 등)**: 문장 단위 dense vector 생성.

## 2. 카테고리별 정리

### 2-1. 동시 행동 기반 협업 필터링 (Scala Spark + Dataproc)

**Dataproc**: Google Cloud 의 매니지드 Hadoop/Spark 서비스. Scala 2.11 + Spark 2.4.4 워크플로로 동작.

| 엔진                     | 설명                                                            |
| ------------------------ | --------------------------------------------------------------- |
| **vt**                   | 사용자 동시 조회 패턴으로 상품 간 유사도. 일반 투어/TNA         |
| **vt-acc / vt-mi / vt-pkg** | VT 의 숙소/민박/패키지 변형 (gid 기반)                        |
| **st / st-acc**          | 동시 검색 기반. 일반 / 숙소                                     |
| **bt**                   | 동시 구매 기반 (VT fork)                                        |
| **ct**                   | 도시 페이지 액션 기반 (BT fork)                                 |
| **rc**                   | 렌터카 VT 변형                                                  |
| **vtf / vtf-cp**         | 항공 구매자가 같이 본 상품 추천 + 도시×카테고리×product_type 단위 cross-product 집계 |
| **vt-deep**              | wide&deep, two-tower, DLRM, tab-transformer 등 딥러닝 14종 (AWS p2/g5 + TensorFlow/PyTorch) |

**vt-deep 의 14종 후보 모델** (일반 산업 라인업):
- Wide & Deep (Google 2016) — linear + DNN 결합, memorization + generalization.
- Two-Tower (YouTube 2019) — user tower + item tower, dot-product score, retrieval 단계.
- DLRM (Meta 2019) — categorical embedding + dense feature → interaction → MLP.
- Tab-Transformer (Amazon 2020) — 정형 데이터용 transformer.
- 그 외 DeepFM, xDeepFM, DCN-v2, AutoInt 등 변형 가능성.

### 2-2. Category Best — 도시×카테고리 인기 (BigQuery + Airflow)

**BigQuery**: Google Cloud 의 서버리스 OLAP (Online Analytical Processing, 분석용) 데이터 웨어하우스. SQL 만으로 페타바이트급 집계 가능.
**Airflow**: Apache 의 워크플로 오케스트레이터. DAG 단위로 실행 스케줄링.

| 엔진           | 설명                                                                       |
| -------------- | -------------------------------------------------------------------------- |
| **cb**         | 일반 offer. 행동 가중합 (`reservation×100 + click×20 + addwish×10 + pageview×1`) |
| **cb2**        | TNA 카테고리1+2 통합                                                       |
| **cb-acc / cb-mi / cb-pkg / cb-vtf / cb-com** | 숙소 / 민박 / 패키지 / 항공 cross / 커뮤니티 콘텐츠 |
| **urb**        | 월간 CTR 피처 테이블 기반 도시×카테고리 (cb 의 CTR 강화 변형)              |

### 2-3. 위치/거리 기반 (Geo-aware)

ES (Elasticsearch) 의 `ST_DISTANCE` 또는 BigQuery 의 `ST_DISTANCE` UDF 활용.

| 엔진                  | 설명                                              |
| --------------------- | ------------------------------------------------- |
| **cross-nearby**      | 숙소 → 근처 TNA cross 추천                        |
| **nearby-tna**        | TNA 상품 간 근접                                  |
| **nearby-products**   | main/detail 용 통합 nearby (주로 통합 숙박 풀)    |
| **localstay-nearby**  | 장기 숙박 간 근접 + 장기 CTR 보정                 |
| **lb / ldp**          | 랜드마크 인기 offer / 도시별 랜드마크 인기도      |

### 2-4. 베스트 / 인기 / 시즌

| 엔진                          | 설명                                                       |
| ----------------------------- | ---------------------------------------------------------- |

### 2-5. 콘텐츠 / NLP

| 엔진                                       | 설명                                                                                  |
| ------------------------------------------ | ------------------------------------------------------------------------------------- |
| **morelike-com / morelike-offer**          | Sentence-BERT (KoBERT/Electra/RoBERTa/BART) 로 제목+서브텍스트 임베딩 → 코사인 유사도. ES MoreLikeThis 대체. `src/morelike/main.py` 는 두 디렉토리에서 **동일**, DAG 만 분리 |
| **rs**                                     | 형태소 분석 + `search_term` 공기빈도 (co-occurrence) 로 연관 검색어                   |
| **nero**                                   | KLUE-BERT NER 추론 서버 (Flask + waitress, AWS GPU AMI). **추천 엔진이 아닌 검색 인프라** |

**임베딩 모델 풀이**:
- **KoBERT**: SKT 가 공개한 한국어 BERT (Bidirectional Encoder Representations from Transformers).
- **Sentence-BERT**: 문장 임베딩에 특화된 BERT 변형 (Siamese 구조).
- **Electra**: BERT 대비 효율적 사전학습 (Generator + Discriminator).
- **RoBERTa**: BERT 의 사전학습 레시피 개선 (Robustly Optimized BERT Approach).
- **BART**: Bidirectional + Auto-Regressive Transformer (Facebook), seq2seq 사전학습.
- **KLUE-BERT**: 한국어 자연어 이해 벤치마크 (Korean Language Understanding Evaluation) 용 BERT.

## 5. 학습 관점 ← /study:* 파이프라인 진입용

본 카탈로그는 study/temp.md #20 (추천 모델링 알고리즘) 의 **입력 자료** 다. 학습 시점에 다음 축으로 분해할 수 있다:

### Phase 1 — CF (협업 필터링) 기초
- 동시 조회 (VT) / 검색 (ST) / 구매 (BT) 의 **공출현 행렬** 구성.
- Item-Item CF vs User-Item CF — OTA 가 Item-Item 채택한 이유 (cold-user 회피).
- Jaccard / Cosine / PMI (Pointwise Mutual Information, 점별 상호정보량) 같은 유사도 메트릭.
- Spark 의 `flatMap + groupByKey` 패턴으로 공출현 카운팅.

### Phase 2 — 베스트 랭킹 (CTR/구매 가중합)
- 행동 가중치 (`reservation×100`) 설계 — **prior knowledge encoding** 관점.
- CTR vs CVR (Conversion Rate, 전환율) — 무엇을 최적화할지에 따라 가중치가 달라진다.
- Wilson score / Bayesian smoothing — 적은 노출 상품의 점수 왜곡 방지.

### Phase 3 — 거리/위치 기반
- Geohash / S2 / H3 — 위경도 → 셀 단위 클러스터링.
- ES `geo_distance` query / BigQuery `ST_DWITHIN` / PostGIS — 인덱스 전략.
- 숙소→TNA cross 추천에서의 **거리 패널티 + 인기 보정** 결합.

### Phase 4 — NLP 임베딩 추천
- Sentence-BERT 두 문장 코사인 유사도가 ES MoreLikeThis (TF-IDF 기반) 보다 우월한 시나리오.
- FAISS / HNSW (Hierarchical Navigable Small World, 계층적 탐색 가능 소세계) — 임베딩 ANN (Approximate Nearest Neighbor, 근사 최근접 이웃) 인덱스.
- Embedding refresh 주기 vs 신상품 cold-start trade-off.

### Phase 5 — 딥러닝 추천 (vt-deep 계열)
- **Wide & Deep**: linear part 가 외운 (memorize) feature interaction + DNN part 가 일반화.
- **Two-Tower**: retrieval 단계용. user tower vs item tower 분리 → 서빙 시 ANN 으로 빠른 매칭.
- **DLRM**: dense + sparse feature 분리, sparse 는 embedding lookup, interaction = dot product.
- **Tab-Transformer**: categorical embedding 에 self-attention 추가.
- **Two-stage retrieval + ranking**: retrieval (Two-Tower) → ranking (DLRM / Wide&Deep) 의 산업 표준 파이프라인.

### Phase 6 — 메타 / 디스트리뷰션
- Section preference (`stb`) — bandit (multi-armed bandit, 다중 슬롯머신 강화학습) 패러다임 적용 후보.
- Default preference (`c2dp`) — cold-start fallback.
- Score boosting (`unionstay-score`) — business rule injection.

### Phase 7 — msa 코드베이스 grounding (msa 적용 후보)
- MSA `analytics` 서비스가 이벤트 수집 + 스코어 산출 (Kafka Streams + ClickHouse). OTA 의 BigQuery + Airflow 패턴과 비교.
- MSA `search` 서비스가 BM25 기반 검색. Hybrid Search (BM25 + dense vector) 도입 시 #19 와 연결.
- MSA `experiment` 서비스가 A/B 테스트 플랫폼. 추천 알고리즘 비교 실험에 활용 가능.
- 향후 ADR 후보: "MSA 에 추천 서비스 도입 — 1단계 CB (Category Best) 룰 기반 → 2단계 CF Spark 잡 → 3단계 임베딩 ANN".
