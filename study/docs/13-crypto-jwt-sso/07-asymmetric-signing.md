# 07. 비대칭 암호 / 디지털 서명

## RSA

- 정수 인수분해 어려움에 기반
- 같은 키로 암호화도, 서명도 가능
- **권장 키 길이**: 2048b 이상 (2030년대까지), 새 시스템은 3072b
- 단점: 키 크기 큼, 연산 느림

### RSA Padding — 절대 raw RSA 쓰지 말 것

| 패딩 | 용도 | 평가 |
|---|---|---|
| PKCS#1 v1.5 (encryption) | 암호화 | **취약** (Bleichenbacher 공격) |
| PKCS#1 v1.5 (signature) | 서명 | 호환성으로 사용 OK |
| **OAEP** | 암호화 | **권장** |
| **PSS** | 서명 | **권장** (확률적, IND-CCA2) |

Raw RSA는 결정론적이라 같은 평문이 같은 암호문이 되고, padding oracle 공격에 취약하다.

## ECDSA (Elliptic Curve DSA)

- 타원곡선 이산로그 어려움 기반
- **키 길이가 훨씬 작음** — ECC-256이 RSA-3072 수준 보안
- 표준 곡선: secp256r1 (P-256), secp384r1
- **약점**: nonce(k) 재사용 시 개인키 즉시 복원 (PS3 해킹, Bitcoin 사고)

## EdDSA / Ed25519

- 결정론적 nonce — k 재사용 위험 원천 차단
- 빠르고 안전, 부수 채널 강함
- SSH 키, Git 서명, TLS 1.3, JWT EdDSA에서 채택 증가

## 비교 (보안 강도 ~128비트 기준)

| 알고리즘 | 키 길이 | 서명 크기 | 검증 속도 |
|---|---|---|---|
| RSA-3072 | 3072b | 384B | 빠름 |
| ECDSA P-256 | 256b | 64B | 빠름 |
| Ed25519 | 256b | 64B | 매우 빠름 |

## 핵심 포인트

- 새 시스템: **Ed25519 우선**
- 호환 필요 시: ECDSA P-256
- 인증서 발급 등 PKI 호환 위해 RSA 유지하는 경우도 있음
- **암호화 vs 서명 키는 분리**하는 것이 원칙 (key separation)

## 개념 관계도

```
                    ┌─────────────────────────────┐
                    │  비대칭 (느림)              │
                    │  - 키 교환                  │
                    │  - 서명 (RSA-PSS, ECDSA,    │
                    │    Ed25519)                 │
                    │  - 인증서                   │
                    └──────────────┬──────────────┘
                                   │ 교환
                                   ▼
                    ┌─────────────────────────────┐
                    │  대칭 (빠름) — AES-256-GCM  │
                    │  - 평문 ↔ 암호문 + 태그     │
                    │  - IV 매번 갱신             │
                    └──────────────┬──────────────┘
                                   │
                                   ▼
                    ┌─────────────────────────────┐
                    │  무결성 단독 — HMAC          │
                    │  (length-ext 방어 구조)      │
                    └──────────────┬──────────────┘
                                   │
                                   ▼
                    ┌─────────────────────────────┐
                    │  해시 SHA-2 / SHA-3 / BLAKE  │
                    └──────────────┬──────────────┘
                                   │ + salt + slow KDF
                                   ▼
                    ┌─────────────────────────────┐
                    │  비밀번호 해싱 — argon2id    │
                    └─────────────────────────────┘
```

**의존 관계**
- HMAC ⊂ Hash (해시 위에 키 결합)
- KDF (PBKDF2/argon2) ⊂ Hash + 비용 함수
- 디지털 서명 = Hash + 비대칭 (메시지 → 해시 → 비대칭 서명)
- AEAD = 대칭 암호화 + 무결성 (CTR + GMAC 합)
